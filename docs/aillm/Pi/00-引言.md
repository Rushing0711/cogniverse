## 0 前置条件

### 0.1 系统要求

| 要求 | 详情 |
|------|------|
| Node.js | 必需（Termux 需 nodejs 包）|
| Windows | 需 bash（推荐 Git Bash）|
| 终端 | Kitty/iTerm2 开箱即用；Ghostty/WezTerm/Alacritty/Windows Terminal 需少量配置 |

> **注意**：Pi 是极简的终端编码代理（minimal terminal coding harness），由 Earendil Inc. 开发，MIT 许可证开源。与 Claude Code（绑定 Anthropic）、Codex（绑定 OpenAI）不同，Pi 不绑定特定模型，支持 15+ provider。核心理念是「Primitives, not features」（提供原语，而非功能）--故意不内置 MCP、sub-agents、权限弹窗、plan mode、to-dos、后台 bash，而是通过 extensions/skills/packages 实现。

### 0.2 Pi 是什么

Pi 的核心特性：

- **极简核心 + 强扩展**：核心只提供原语，工作流特定行为推到 extensions/skills/packages
- **模型无关**：支持 15+ provider（Anthropic、OpenAI、Google、Bedrock、Azure、Mistral、Groq、Cerebras、xAI、Hugging Face、Kimi、MiniMax、NVIDIA、OpenRouter、Ollama 等）
- **四运行模式**：交互式 TUI、Print/JSON、RPC、SDK
- **树形会话**：单文件存所有分支，`/tree` 原地探索替代方案
- **热重载**：`/reload` 无需重启应用扩展改动
- **Skills 跨 harness 兼容**：实现 Agent Skills 标准，可复用 Claude Code/Codex skills
- **OAuth 订阅登录**：支持 ChatGPT Plus/Pro、Claude Pro/Max、GitHub Copilot

## 1 安装 Pi

安装和配置 Pi 非常简单，详细步骤请参考官方文档：

https://pi.dev/docs/latest

### 1.1 官方安装脚本（推荐）

**Linux / macOS**：

```bash
$ curl -fsSL https://pi.dev/install.sh | sh
```

### 1.2 npm

```bash
$ npm install -g --ignore-scripts @earendil-works/pi-coding-agent
```

> **注意**：`--ignore-scripts` 禁用依赖生命周期脚本，Pi 正常安装不需要 install scripts。

### 1.3 pnpm / bun

```bash
$ pnpm install -g --ignore-scripts @earendil-works/pi-coding-agent
$ bun install -g --ignore-scripts @earendil-works/pi-coding-agent
```

### 1.4 验证与更新

```bash
$ pi --version
pi 0.x.x

# 更新 pi 本身
$ pi update --self

# 更新 pi 和所有包
$ pi update --all

# 强制重装
$ pi update --self --force
```

### 1.5 卸载

用对应包管理器卸载（curl 安装的也用 npm 卸载）：

```bash
$ npm uninstall -g @earendil-works/pi-coding-agent
```

<span style="color:red;font-weight:bold;">卸载后 `~/.pi/agent/` 下的设置、凭证、会话、packages 保留</span>，想完全清理需手动删除：

```bash
$ rm -rf ~/.pi
```

## 2 登录认证

Pi 支持多种认证方式，选择合适的方式直接影响可用 Provider 和计费。

### 2.1 认证方式对比

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| OAuth 订阅登录 | 用 ChatGPT/Claude/Copilot 订阅额度 | 订阅用户 |
| API Key 环境变量 | 设置 `ANTHROPIC_API_KEY` 等 | 脚本、CI/CD |
| `/login` 命令 | 交互式选择 provider，存入 `auth.json` | 首次配置 |

### 2.2 OAuth 订阅登录

在交互模式中运行 `/login`，选择 provider：

| Provider | 订阅要求 | 说明 |
|----------|----------|------|
| ChatGPT Plus/Pro (Codex) | ChatGPT Plus/Pro | OpenAI 官方背书（Codex for OSS）|
| Claude Pro/Max | Claude Pro/Max | 第三方 harness 使用会从 extra usage 扣费，按 token 计费 |
| GitHub Copilot | Copilot 订阅 | 支持 GitHub Enterprise Server |

`/logout` 清除凭证。token 存储在 `~/.pi/agent/auth.json`，自动刷新。

### 2.3 API Key 环境变量

设置环境变量后启动：


### 2.5 凭据存储位置

| 平台 | 路径 | 权限 |
|------|------|------|
| 所有平台 | `~/.pi/agent/auth.json` | 0600（仅用户可读写）|

token 自动刷新，活跃会话通常不需要重新登录。

### 2.6 Key 字段高级特性

Key 字段支持高级特性，便于从密码管理器读取密钥：

| 特性 | 语法 | 说明 |
|------|------|------|
| Shell 命令 | `!command` | 执行命令取 stdout 作为 key |
| 环境变量插值 | `$ENV_VAR` 或 `${ENV_VAR}` | 从环境变量读取 |
| 字面量 | `sk-xxx` | 直接字符串 |
| 转义 | `$$`、`$!` | 转义特殊字符 |

示例--从 macOS Keychain 读取：

```json
{
  "anthropic": "!security find-generic-password -s anthropic -w"
}
```

从 1Password 读取：

```json
{
  "anthropic": "!op read 'op://Private/Anthropic/api-key'"
}
```

**凭证解析顺序**：CLI `--api-key` -> `auth.json` -> 环境变量 -> `models.json` 自定义 provider keys。

## 3 pi 命令速查

### 3.1 常用命令

```bash
$ pi                             # 启动交互式 TUI（默认）
$ pi "List all .ts files in src/" # 带初始 prompt 启动
$ pi -p "Summarize this codebase" # 非交互，打印响应并退出
$ pi @README.md "总结这个"        # 引用文件
$ pi install <source>            # 安装包
$ pi remove <source>             # 移除包（别名 uninstall）
$ pi update --self               # 更新 pi 本身
$ pi update --all                # 更新 pi 和所有包
$ pi list                        # 列出已安装包
$ pi config                      # 启用/禁用包资源
```

### 3.2 子命令完整清单

| 子命令 | 说明 |
|--------|------|
| `pi [options] [@files...] [messages...]` | 启动 TUI（默认）|
| `pi install <source> [-l]` | 安装包，`-l` 项目级 |
| `pi remove <source> [-l]` | 移除包（别名 `uninstall`）|
| `pi update [source\|self\|pi\|all]` | 更新 pi 或单个包 |
| `pi update --extensions` | 仅更新包 |
| `pi update --extension <src>` | 更新单个包 |
| `pi list` | 列出已安装包 |
| `pi config` | 启用/禁用包资源 |

### 3.3 常用全局 flag

**模式**：

| Flag | 说明 |
|------|------|
| `-p` / `--print` | 打印响应并退出（非交互）|
| `--mode json` | 输出所有事件为 JSON 行 |
| `--mode rpc` | RPC 模式（stdin/stdout JSONL）|
| `--export <in> [out]` | 导出会话为 HTML |

**模型**：

| Flag | 说明 |
|------|------|
| `--provider <name>` | provider 名 |
| `--model <pattern>` | 模型 ID 或模式，支持 `provider/id` 和 `:thinking` 后缀 |
| `--api-key <key>` | 覆盖环境变量 |
| `--thinking <level>` | `off`/`minimal`/`low`/`medium`/`high`/`xhigh`/`max` |
| `--models <patterns>` | 逗号分隔，Ctrl+P 循环模型 |
| `--list-models [search]` | 列出可用模型 |

**会话**：

| Flag | 说明 |
|------|------|
| `-c` / `--continue` | 继续最近会话 |
| `-r` / `--resume` | 浏览选择会话 |
| `--session <path\|id>` | 指定会话 |
| `--fork <path\|id>` | 派生会话 |
| `--session-dir <dir>` | 自定义会话目录 |
| `--no-session` | 临时模式不保存 |
| `--name <name>` / `-n` | 设置会话显示名 |

**工具**：

| Flag | 说明 |
|------|------|
| `--tools <list>` / `-t` | 白名单工具 |
| `--exclude-tools <list>` / `-xt` | 禁用工具 |
| `--no-builtin-tools` / `-nbt` | 禁用内置工具但保留扩展工具 |
| `--no-tools` / `-nt` | 禁用所有工具 |

**资源**：

| Flag | 说明 |
|------|------|
| `-e` / `--extension <source>` | 加载扩展（路径/npm/git，可重复）|
| `--skill <path>` | 加载 skill |
| `--prompt-template <path>` | 加载 prompt 模板 |
| `--theme <path>` | 加载主题 |
| `--no-extensions` / `--no-skills` / `--no-prompt-templates` / `--no-themes` | 禁用发现 |
| `--no-context-files` / `-nc` | 禁用 AGENTS.md/CLAUDE.md |

**其他**：

| Flag | 说明 |
|------|------|
| `--system-prompt <text>` | 替换默认系统提示 |
| `--append-system-prompt <text>` | 追加系统提示 |
| `--verbose` | 详细输出 |
| `-a` / `--approve` | 批准项目信任 |
| `-na` / `--no-approve` | 拒绝项目信任 |
| `--offline` | 禁用启动时网络操作 |
| `-h` / `--help` | 帮助 |
| `-v` / `--version` | 版本 |

### 3.4 文件参数

`@` 前缀引用文件：

```bash
$ pi @README.md "总结这个文件"
$ pi -p @screenshot.png "图里是什么"
```

### 3.5 示例

```bash
$ pi "List all .ts files in src/"            # 交互式带初始 prompt
$ pi -p "Summarize this codebase"            # 非交互
$ cat README.md | pi -p "Summarize this"     # 管道输入
$ pi --model openai/gpt-4o "Help me"         # provider/id 格式
$ pi --model sonnet:high "Solve this"        # thinking 简写
$ pi --tools read,grep,find,ls -p "Review"   # 只读模式
$ pi --thinking high "Solve this"            # 高 thinking
```
