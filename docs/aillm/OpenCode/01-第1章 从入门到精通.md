# 从入门到精通

## 1 模型与 Provider

OpenCode 支持 75+ LLM 提供商，不绑定特定模型，这是它与 Claude Code（仅 Anthropic）和 Codex（主推 OpenAI）的最大差异。

### 1.1 75+ Provider 概览

基于 AI SDK 和 Models.dev，大多数热门提供商默认预加载：

| 类别 | Provider |
|------|----------|
| 头部厂商 | Anthropic、OpenAI、Google Vertex AI、Amazon Bedrock、Azure OpenAI |
| 聚合器 | OpenRouter、OpenCode Zen、Helicone、Vercel AI Gateway、Cloudflare AI Gateway |
| GitHub | GitHub Copilot（OAuth）|
| GitLab | GitLab Duo（OAuth/PAT）|
| 国内/亚洲 | DeepSeek、MiniMax、Moonshot AI（Kimi）、Z.AI（GLM）、302.AI |
| 开源/本地 | Ollama、LM Studio、llama.cpp、Ollama Cloud |
| 推理优化 | Groq、Cerebras、Together AI、Fireworks AI、Deep Infra |
| 其他 | xAI（Grok）、Hugging Face、Venice AI、Baseten、Cortecs 等 |

### 1.2 模型 ID 格式

`provider_id/model_id`，例如：

```
anthropic/claude-sonnet-4-5
opencode/gpt-5.1-codex          # Zen 网关
openrouter/google/gemini-2.5-flash
openai/gpt-5.4
ollama/qwen3-coder
```

```bash
# 启动时指定
$ opencode -m anthropic/claude-sonnet-4-5

# config 中指定
{
  "model": "anthropic/claude-sonnet-4-5"
}
```

### 1.3 模型加载优先级

从高到低：

1. `--model` / `-m` 命令行标志
2. 配置中的 `model` 字段
3. 上次使用的模型
4. 按内部优先级排列的第一个可用模型

### 1.4 small_model 轻量任务模型

`small_model` 用于轻量任务（如生成会话标题、摘要），节省成本：

```jsonc
{
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5"
}
```

### 1.5 opencode models 命令

```bash
$ opencode models                    # 所有提供商
$ opencode models anthropic          # 指定提供商
$ opencode models --refresh          # 刷新缓存
$ opencode models --verbose          # 详细信息（含费用）
```

### 1.6 OpenCode Zen 精选网关

OpenCode Zen 是 OpenCode 团队测试验证的精选模型网关，推荐入门：

- 在 TUI 中 `/connect` 选择 `opencode`
- 前往 opencode.ai/auth 获取密钥
- 按请求付费，支持自动充值（余额低于 $5 时自动充值 $20）
- 团队功能：Admin/Member 角色、模型访问控制、自带密钥（BYOK）

```bash
$ opencode auth login
# 选择 opencode，粘贴 Zen 密钥

# 使用 Zen 模型
$ opencode -m opencode/gpt-5.1-codex
```

### 1.8 自定义 OpenAI 兼容 Provider

接入任意 OpenAI 兼容 API：

```jsonc
{
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI Provider",
      "options": {
        "baseURL": "https://api.myprovider.com/v1",
        "apiKey": "{env:MY_API_KEY}",
        "headers": { "Authorization": "Bearer custom-token" }
      },
      "models": {
        "my-model-name": {
          "name": "My Model Display Name",
          "limit": { "context": 200000, "output": 65536 }
        }
      }
    }
  }
}
```

**npm 包选择**：

| 端点 | npm 包 |
|------|--------|
| `/v1/chat/completions` | `@ai-sdk/openai-compatible` |
| `/v1/responses` | `@ai-sdk/openai` |

```jsonc
{
```

```jsonc
    "bedrock": {
      "options": {
```

### 1.11 模型变体（Variants）

支持为同一模型配置不同设置：

```jsonc
{
  "provider": {
    "openai": {
      "models": {
        "gpt-5": {
          "variants": {
            "thinking": { "reasoningEffort": "high", "textVerbosity": "low" },
            "fast": { "disabled": true }
          }
        }
      }
    }
  }
}
```

切换变体快捷键：`ctrl+t`（`variant_cycle`）。详见第 13 章。

## 2 权限模式

OpenCode 用细粒度权限系统控制 agent 的操作范围，支持 glob 模式匹配，比 Claude Code 的 permissions 更灵活。

### 2.1 三种操作类型

| 操作 | 行为 |
|------|------|
| `allow` | 无需审批直接运行 |
| `ask` | 提示审批 |
| `deny` | 阻止操作 |

### 2.2 配置方式

#### 全局字符串

```jsonc
{
  "permission": "allow"    // 所有操作默认允许
}
```

#### 按工具

```jsonc
{
  "permission": {
    "*": "ask",
    "bash": "allow",
    "edit": "deny"
  }
}
```

#### 细粒度 glob 对象

```jsonc
{
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "npm *": "allow",
      "rm *": "deny",
      "grep *": "allow"
    },
    "edit": {
      "*": "deny",
      "packages/web/src/content/docs/*.mdx": "allow"
    }
  }
}
```

### 2.3 通配符规则

| 符号 | 说明 |
|------|------|
| `*` | 零个或多个任意字符 |
| `?` | 精确一个字符 |
| `~` 或 `$HOME` | 主目录展开 |

### 2.4 可用权限表

| 权限 | 说明 | 匹配对象 |
|------|------|----------|
| `read` | 读取文件 | 文件路径 |
| `edit` | 所有文件修改（edit/write/patch）| 文件路径 |
| `glob` | 文件通配 | 通配模式 |
| `grep` | 内容搜索 | 正则表达式 |
| `bash` | 运行 shell 命令 | 解析后的命令 |
| `task` | 启动子代理 | 子代理类型 |
| `skill` | 加载技能 | 技能名称 |
| `lsp` | 运行 LSP 查询 | 不支持细粒度 |
| `webfetch` | 获取 URL | URL |
| `websearch` | 网页搜索 | 查询内容 |
| `external_directory` | 访问工作目录外路径 | 路径 |
| `doom_loop` | 工具调用重复 3 次 | - |

### 2.5 默认值与 .env 保护

- 大多数权限默认 `allow`
- `doom_loop` 和 `external_directory` 默认 `ask`
- `read` 默认 `allow`，但 <span style="color:red;font-weight:bold;">`.env` 文件默认 `deny`</span>（`*.env.example` 例外）

### 2.6 Ask 选项

弹出审批时，选项：

| 选项 | 行为 |
|------|------|
| `once` | 仅批准本次 |
| `always` | 批准匹配模式的后续请求（当前会话有效）|
| `reject` | 拒绝 |

### 2.7 代理级权限覆盖

为特定代理覆盖全局权限：

```jsonc
{
  "permission": {
    "bash": { "*": "ask", "git *": "allow" }
  },
  "agent": {
    "build": {
      "permission": {
        "bash": {
          "*": "ask",
          "git commit *": "ask",
          "git push *": "deny"
        }
      }
    }
  }
}
```

### 2.8 Policies（实验性）

控制是否可使用某资源（如 LLM 提供商），与 permissions 分离：

```jsonc
{
  "experimental": {
    "policies": [
      { "effect": "deny", "action": "provider.use", "resource": "openai" },
      { "effect": "allow", "action": "provider.use", "resource": "anthropic" }
    ]
  }
}
```

规则按顺序评估，最后匹配的优先。全局策略优先于项目策略。

### 2.9 OPENCODE_PERMISSION 环境变量

```bash
# 内联 JSON 权限配置
$ OPENCODE_PERMISSION='{"bash":"ask","edit":"deny"}' opencode
```

适合 CI/CD 场景，临时覆盖权限。

### 2.10 常见权限配置场景

#### 场景1：只读浏览

```jsonc
{
  "permission": {
    "*": "ask",
    "read": "allow",
    "glob": "allow",
    "grep": "allow"
  }
}
```

#### 场景2：自动编辑但命令需审批

```jsonc
{
  "permission": {
    "edit": "allow",
    "bash": { "*": "ask", "git *": "allow", "npm *": "allow" },
    "read": "allow"
  }
}
```

#### 场景3：保护特定文件

```jsonc
{
  "permission": {
    "edit": {
      "*": "allow",
      "**/.env": "deny",
      "**/secrets.json": "deny",
      "**/*.key": "deny"
    }
  }
}
```

## 3 基本交互（TUI）

### 3.1 启动与基本操作

```bash
$ opencode                    # 在当前目录启动 TUI
$ opencode /path/to/project   # 指定工作目录
$ opencode -m anthropic/claude-sonnet-4-5   # 指定模型
$ opencode -c                 # 继续上一个会话
$ opencode -s <session-id>    # 指定会话 ID
```

启动后进入交互式 TUI，底部是输入框，上方是对话区和工具调用展示。

### 3.2 前导键机制

OpenCode 大多数快捷键需要先按前导键 `ctrl+x`，再按对应键：

```
ctrl+x n    # 新会话
ctrl+x l    # 会话列表
ctrl+x c    # 压缩会话
```

前导键超时默认 2000ms，可在 `tui.json` 中配置：

```jsonc
{
  "leader_timeout": 2000
}
```

如果 2000ms 内未按下一个键，前导键失效。

### 3.3 文件引用 @

输入 `@` 触发模糊搜索文件：

```
看一下 @src/utils/parser.ts 的实现
```

`@` 后跟文件路径模糊匹配，选中后文件内容作为上下文注入。

### 3.4 Bash 命令 !

输入 `!command` 执行 shell 命令，输出注入到 prompt：

```
基于 !git log --oneline -10 的最近提交，生成 release notes
```

`!` 后跟 shell 命令，OpenCode 执行后把输出作为上下文。

### 3.5 图片支持

终端支持的话（如 WezTerm、Ghostty），可拖放图片到终端作为视觉上下文：

- 拖放图片文件到终端窗口
- 图片作为多模态输入传给支持的模型（如 Claude、GPT-4o）

### 3.6 Tab 切换主代理

| 快捷键 | 说明 |
|--------|------|
| `Tab` | 切换主代理（Build / Plan）|
| `Shift+Tab` | 反向切换 |

**Build 代理**：默认代理，启用所有工具，用于开发工作。

**Plan 代理**：只读代理，分析代码、建议更改，file edits 和 bash 默认 ask。

### 3.7 F2 / ctrl+t 切换模型与变体

| 快捷键 | 说明 |
|--------|------|
| `F2` / `Shift+F2` | 循环最近使用的模型 |
| `ctrl+t` | 切换模型变体（variant_cycle）|

变体控制推理能力（如 thinking、fast），详见第 13 章。

### 3.8 滚动与消息操作快捷键

| 快捷键 | 说明 |
|--------|------|
| `PageUp` / `PageDown` | 翻页 |
| `ctrl+alt+u` / `ctrl+alt+d` | 半页滚动 |
| `ctrl+g` / `home` | 第一条 |
| `ctrl+alt+g` / `end` | 最后一条 |
| `ctrl+x y` | 复制消息 |
| `ctrl+x u` | 撤销（`/undo`）|
| `ctrl+x r` | 重做（`/redo`）|
| `ctrl+x h` | 切换隐藏 |

### 3.9 编辑器快捷键（Readline/Emacs 风格）

| 快捷键 | 说明 |
|--------|------|
| `ctrl+a` / `ctrl+e` | 行首 / 行尾 |
| `ctrl+b` / `ctrl+f` | 前后移动字符 |
| `alt+b` / `alt+f` | 前后移动单词 |
| `ctrl+k` / `ctrl+u` | 删到行尾 / 行首 |
| `ctrl+w` / `alt+d` | 删除前 / 后单词 |
| `ctrl+d` | 删除光标字符 |
| `ctrl+t` | 交换前后字符 |
| `ctrl+g` | 取消弹出窗口 |

### 3.10 好 Prompt 公式与修正方向

清晰的 prompt 结构：

```
[上下文] + [任务] + [约束] + [期望输出格式]
```

示例：

```
我在做一个 VitePress 知识库（见 docs/.vitepress/config.mjs），
需要新增 OpenCode 文档目录 docs/aillm/OpenCode/。
约束：仿照 docs/aillm/Codex/ 的风格，中文为主，代码标识符英文。
请先列出 00-引言.md 的章节大纲，不要直接写代码。
```

**修正方向**：如果 OpenCode 走偏了，用 `escape` 中断当前 turn，或直接发送纠正消息：

```
# OpenCode 正在过度抽象
停，不要引入 BaseAgent 抽象层，直接在现有函数上改

# OpenCode 正在改错文件
这个改动应该在 src/utils.ts，不是 src/core.ts
```

## 4 AGENTS.md 项目指令

`AGENTS.md` 是 OpenCode 的项目自定义指令文件，遵循开放标准 [agents.md](https://agents.md)。OpenCode 同时兼容 Claude Code 的 `CLAUDE.md` 作为回退。

### 4.1 AGENTS.md 该放什么

```markdown
# 项目名称

## 常用命令
- `pnpm docs:dev` - 启动开发服务器
- `pnpm docs:build` - 构建生产版本

## 代码风格
- 文档使用中文
- 代码标识符保持英文
- 配置示例用 JSON/JSONC

## 项目规范
- 一定不要删除文档 *.md 中的图片
- 构建产物 docs/.vitepress/dist 不提交到 git

## 重要提醒
- 包管理器必须用 pnpm
- 端口 8750 用于本地开发
```

### 4.2 /init 自动生成

在 TUI 中运行 `/init`，OpenCode 会扫描当前项目并生成 `AGENTS.md` 脚手架：

```
/init
```

或用快捷键 `ctrl+x i`。生成的脚手架包含项目结构、常用命令、代码风格等基础信息，用户再手动完善。

### 4.3 兼容 CLAUDE.md 回退机制

OpenCode 兼容 Claude Code 的指令文件：

- 优先读取 `AGENTS.md`
- 若不存在，回退读取 `CLAUDE.md`

这意味着从 Claude Code 迁移到 OpenCode 时，<span style="color:red;font-weight:bold;">无需立即重命名 CLAUDE.md</span>，OpenCode 会自动兼容。

禁用 CLAUDE.md 回退：

```bash
$ export OPENCODE_DISABLE_CLAUDE_CODE_PROMPT=1
$ opencode
```

### 4.4 instructions 自定义指令文件

除了 `AGENTS.md`，还可以引用其他指令文件：

```jsonc
{
  "instructions": [
    "CONTRIBUTING.md",
    "docs/*.md",
    "https://example.com/team-guide.md"
  ]
}
```

- 支持相对路径、glob 模式、远程 URL
- 文件内容作为指令注入到系统 prompt

### 4.5 全局 vs 项目级指令

| 层级 | 位置 | 用途 |
|------|------|------|
| 全局 | `~/.config/opencode/AGENTS.md` | 用户全局偏好 |
| 项目 | `<repo>/AGENTS.md` 或 `<repo>/CLAUDE.md` | 项目特定规则 |

全局指令适合放：

```markdown
# ~/.config/opencode/AGENTS.md
## Working agreements
- Always run `npm test` after modifying JavaScript files.
- Prefer `pnpm` when installing dependencies.
- Ask for confirmation before adding new production dependencies.
```

### 4.6 禁用 Claude Code 兼容

完全禁用所有 `.claude` 支持：

```bash
$ export OPENCODE_DISABLE_CLAUDE_CODE=1
$ opencode
```

细粒度禁用：

| 环境变量 | 说明 |
|----------|------|
| `OPENCODE_DISABLE_CLAUDE_CODE` | 禁用所有 .claude 支持 |
| `OPENCODE_DISABLE_CLAUDE_CODE_PROMPT` | 仅禁用 `~/.claude/CLAUDE.md` |
| `OPENCODE_DISABLE_CLAUDE_CODE_SKILLS` | 仅禁用 `.claude/skills` |

## 5 opencode.json 配置体系

### 5.1 配置文件位置与优先级

从低到高（合并而非替换）：

1. **远程配置**（`.well-known/opencode`）-- 组织默认值
2. **全局配置**（`~/.config/opencode/opencode.json`）-- 用户偏好
3. **自定义配置**（`OPENCODE_CONFIG` 环境变量）
4. **项目配置**（项目根目录 `opencode.json`）-- 项目特定设置
5. **`.opencode` 目录** -- 代理、命令、插件
6. **内联配置**（`OPENCODE_CONFIG_CONTENT` 环境变量）-- 运行时覆盖

```
远程 .well-known/opencode           ← 最低
    ↓
~/.config/opencode/opencode.json    ← 全局
    ↓
$OPENCODE_CONFIG                    ← 自定义路径
    ↓
<repo>/opencode.json                ← 项目级
    ↓
<repo>/.opencode/                   ← 项目扩展（agents/commands/plugins）
    ↓
$OPENCODE_CONFIG_CONTENT            ← 内联覆盖（最高）
```

### 5.2 JSON/JSONC 格式与 $schema

支持 JSON 和 JSONC（带注释的 JSON）：

```jsonc
// ~/.config/opencode/opencode.json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5",
  // 轻量任务用小模型
  "small_model": "anthropic/claude-haiku-4-5"
}
```

`$schema` 字段启用 IDE 自动补全和校验。

### 5.3 核心字段分类

```jsonc
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5",          // 默认模型
  "small_model": "anthropic/claude-haiku-4-5",     // 轻量任务模型
  "default_agent": "build",                          // 默认代理
  "autoupdate": true,                                // 自动更新（true/false/"notify"）
  "share": "manual",                                 // 分享模式（manual/auto/disabled）
  "theme": "opencode",                               // 主题
  "instructions": ["CONTRIBUTING.md", "docs/*.md"], // 指令文件
  "disabled_providers": ["openai"],                  // 禁用提供商
  "enabled_providers": ["anthropic", "openai"],      // 启用提供商白名单
  "provider": { /* 提供商配置 */ },
  "agent": { /* 代理配置 */ },
  "command": { /* 命令配置 */ },
  "mcp": { /* MCP 服务器配置 */ },
  "permission": { /* 权限配置 */ },
  "tools": { /* 工具配置 */ },
  "formatter": { /* 格式化工具 */ },
  "lsp": { /* LSP 服务器 */ },
  "plugin": ["opencode-helicone-session"],           // npm 插件
  "server": { /* 服务器配置 */ },
  "tui": { /* TUI 配置 */ },
  "compaction": { "auto": true, "prune": false, "reserved": 10000 },
  "watcher": { "ignore": ["node_modules/**"] },
  "references": { /* 项目引用 */ },
  "experimental": { /* 实验性功能 */ }
}
```

### 5.4 变量替换

配置文件中支持变量替换：

| 语法 | 说明 |
|------|------|
| `{env:VARIABLE_NAME}` | 环境变量 |
| `{file:path/to/file}` | 文件内容 |

示例：

```jsonc
{
  "provider": {
    "myprovider": {
      "options": {
        "apiKey": "{env:MY_API_KEY}",          // 从环境变量读取
        "instructions": "{file:./prompts.txt}" // 从文件读取
      }
    }
  }
}
```

### 5.5 tui.json 独立配置

TUI 配置独立于 `opencode.json`，位于 `~/.config/opencode/tui.json`：

```jsonc
{
```

## 6 opencode run 非交互模式

`opencode run` 是 OpenCode 的非交互式入口，适合脚本化、CI/CD、批量处理。与 `codex exec` 或 `claude -p` 类似。

### 6.1 基本用法

```bash
# 流式输出到 stdout
$ opencode run "summarize the repository structure"

# 输出重定向到文件
$ opencode run "generate release notes for the last 10 commits" | tee release-notes.md
```

### 6.2 --format json 结构化输出

```bash
$ opencode run --format json "summarize the repo structure" | jq
```

JSON 输出适合用 `jq` 过滤特定字段，或传递给下游程序处理。

### 6.3 --thinking 显示思考块

```bash
$ opencode run --thinking "explain async/await"
```

显示模型的思考/推理过程，适合调试复杂问题。注意：`--thinking` 只控制**显示**，实际推理能力由模型变体控制（见第 13 章）。

### 6.4 --variant 指定推理级别

```bash
$ opencode run --variant high "complex problem"
$ opencode run --variant thinking "explain this algorithm"
```

变体控制模型的推理深度，内置变体如 `high`、`max`、`thinking`、`fast` 等。

### 6.5 --file / -f 附加文件

```bash
$ opencode run -f ./README.md "summarize this file"
$ opencode run -f ./src/index.ts -f ./src/utils.ts "compare these files"
```

可重复 `-f` 附加多个文件作为上下文。

### 6.6 --share 分享会话

```bash
$ opencode run --share "generate a release report"
# 输出会包含分享链接：opncd.ai/s/<share-id>
```

生成可分享的会话链接，详见第 17 章。

### 6.7 --attach 连接到运行中的服务器

```bash
# 先启动服务器
$ opencode serve &

# 连接并运行
$ opencode run --attach "..."
```

适合多个 `run` 命令共享同一后端状态。

### 6.8 --title 会话标题

```bash
$ opencode run --title "release-notes-2026-07" "generate release notes"
```

为会话指定标题，方便后续在 `/sessions` 列表中查找。

### 6.9 CI/CD 集成示例

**基本 CI 任务**：

```bash
$ opencode run --format json "review the changes in this PR" > review.json
```

**GitHub Actions 示例**：

```yaml
- name: Run OpenCode
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    opencode run --format json "review the changes" > opencode-output.json
```

**批量处理**：

```bash
# 处理多个文件
for file in src/*.ts; do
  opencode run --format json -f "$file" "add type annotations" >> output.jsonl
done
```

### 6.10 opencode serve 与 opencode web 对比

| 命令 | 说明 | 适用场景 |
|------|------|----------|
| `opencode run` | 一次性非交互运行 | 脚本、CI/CD |
| `opencode serve` | 启动无界面 HTTP 服务器 | 后端服务、API 集成 |
| `opencode web` | 启动带 Web 界面的服务器 | 团队共享、远程访问 |
| `opencode attach` | 连接到已运行的后端 | 多客户端共享状态 |

```bash
# serve：无界面 API
$ opencode serve --port 4096

# web：带 Web UI
$ opencode web --port 4096 --hostname 0.0.0.0 --mdns

# attach：连接到运行中的服务器
$ opencode attach http://localhost:4096
```

## 7 MCP 集成

MCP（Model Context Protocol）让 OpenCode 连接外部工具和数据源。配置直接写在 `opencode.json` 的 `mcp` 字段下。

### 7.1 本地 MCP 服务器

```jsonc
{
  "mcp": {
    "my-local-mcp": {
      "type": "local",
      "command": ["npx", "-y", "my-mcp-command"],
      "enabled": true,
      "environment": { "MY_ENV_VAR": "value" },
      "timeout": 5000
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `type` | `"local"` |
| `command` | 命令数组，如 `["npx", "-y", "..."]` |
| `enabled` | 是否启用 |
| `environment` | 传给 server 的环境变量 |
| `timeout` | 启动超时（毫秒）|

### 7.2 远程 MCP 服务器

```jsonc
{
  "mcp": {
    "my-remote-mcp": {
      "type": "remote",
      "url": "https://my-mcp-server.com",
      "enabled": true,
      "headers": { "Authorization": "Bearer MY_API_KEY" },
      "oauth": {},
      "timeout": 5000
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `type` | `"remote"` |
| `url` | 远程服务器地址 |
| `headers` | 自定义 HTTP 头 |
| `oauth` | OAuth 配置（或 `false` 禁用）|

### 7.3 OAuth 自动检测与流程

OpenCode 自动检测 OAuth：

- 检测到 401 响应时自动启动 OAuth 流程
- 支持动态客户端注册（RFC 7591）
- Token 存储在 `~/.local/share/opencode/mcp-auth.json`
- 可禁用 OAuth：`"oauth": false`

### 7.4 opencode mcp 命令

```bash
$ opencode mcp add                       # 添加 MCP 服务器
$ opencode mcp list                      # 列出所有 MCP（别名 opencode mcp ls）
$ opencode mcp auth [name]               # OAuth 认证
$ opencode mcp auth list                 # 列出认证状态
$ opencode mcp logout [name]             # 移除 OAuth 凭据
$ opencode mcp debug <name>              # 调试连接问题
```

### 7.5 工具管理（glob 模式）

MCP 工具以服务器名称为前缀注册。可用 glob 模式管理：

```jsonc
{
  "mcp": {
    "my-mcp": {
      "type": "local",
      "command": ["npx", "-y", "my-mcp"]
    }
  },
  "tools": {
    "my-mcp*": false                    // 全局禁用所有 my-mcp 工具
  },
  "agent": {
    "my-agent": {
      "tools": {
        "my-mcp*": true                 // 特定代理启用
      }
    }
  }
}
```

### 7.6 常用 MCP server

| Server | URL/命令 | 说明 |
|--------|----------|------|
| Sentry | `https://mcp.sentry.dev/mcp`（OAuth）| 访问 Sentry 日志 |
| Context7 | `https://mcp.context7.com/mcp` | 连接最新开发者文档 |
| Grep by Vercel | `https://mcp.grep.app` | 代码搜索 |
| Playwright | `npx -y @playwright/mcp` | 浏览器自动化 |
| GitHub | `npx -y @modelcontextprotocol/server-github` | 管理 PR、issues |

安装示例：

```jsonc
{
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com/mcp",
      "enabled": true
    },
    "playwright": {
      "type": "local",
      "command": ["npx", "-y", "@playwright/mcp"],
      "enabled": true
    }
  }
}
```

### 7.7 上下文占用注意事项

<span style="color:red;font-weight:bold;">MCP 服务器会占用上下文空间，建议谨慎选择</span>。特别是 GitHub MCP 服务器消耗较大，长会话中可能导致上下文膨胀。

优化策略：

- 只启用当前任务需要的 MCP server
- 用 `tools` 配置禁用不用的工具
- 长会话中用 `/compact` 压缩上下文

## 8 Agent 与 Subagent

OpenCode 的 Agent 系统分主代理（primary）和子代理（subagent），与 Claude Code 的 Task 工具类似但更灵活。

### 8.1 主代理与子代理的区别

| 类型 | 说明 | 调用方式 |
|------|------|----------|
| 主代理（primary）| 直接交互的主要助手 | `Tab` 键切换 |
| 子代理（subagent）| 主代理调用的专业助手 | `@提及` 或自动调用 |

### 8.2 内置主代理

| 代理 | 说明 |
|------|------|
| **Build** | 默认代理，启用所有工具，用于开发工作 |
| **Plan** | 只读代理，分析代码、建议更改，file edits 和 bash 默认 ask |
| **Compaction** | 隐藏的系统代理，压缩长上下文 |
| **Title** | 隐藏的系统代理，生成会话标题 |
| **Summary** | 隐藏的系统代理，创建会话摘要 |

### 8.3 内置子代理

| 代理 | 说明 |
|------|------|
| **General** | 通用代理，研究复杂问题、多步骤任务，有完整工具访问权限 |
| **Explore** | 快速只读代理，探索代码库，不能修改文件 |
| **Scout** | 只读代理，外部文档和依赖研究，克隆仓库到托管缓存 |

### 8.4 自定义 Agent（JSON 配置）

```jsonc
{
  "agent": {
    "build": {
      "mode": "primary",
      "model": "anthropic/claude-sonnet-4-5",
      "prompt": "{file:./prompts/build.txt}",
      "temperature": 0.3,
      "steps": 10,
      "tools": { "write": true, "edit": true, "bash": true },
      "permission": { /* 代理级权限 */ }
    },
    "code-reviewer": {
      "description": "Reviews code for best practices",
      "mode": "subagent",
      "model": "anthropic/claude-sonnet-4-5",
      "prompt": "You are a code reviewer...",
      "tools": { "write": false, "edit": false },
      "hidden": true,
      "color": "#ff6b6b"
    }
  }
}
```

### 8.5 自定义 Agent（Markdown 配置）

位置：`~/.config/opencode/agents/review.md` 或 `.opencode/agents/review.md`

```markdown
---
description: Reviews code for quality and best practices
mode: subagent
model: anthropic/claude-sonnet-4-5
temperature: 0.1
tools:
  write: false
  edit: false
  bash: false
permission:
  edit: deny
  bash:
    "*": ask
    "git diff": allow
---

You are in code review mode. Focus on:
- Code quality and best practices
- Potential bugs and edge cases
```

### 8.6 配置选项详解

| 字段 | 说明 |
|------|------|
| `description`（必需）| 功能描述，用于自动调用匹配 |
| `mode` | `primary` / `subagent` / `all`（默认 `all`）|
| `model` | 覆盖模型 |
| `prompt` | 系统提示词文件 |
| `temperature` | 0.0-1.0，控制随机性 |
| `top_p` | 控制响应多样性 |
| `steps` | 最大迭代次数（旧名 `maxSteps` 已弃用）|
| `tools` | 工具启用/禁用 |
| `permission` | 代理级权限覆盖 |
| `hidden` | 从自动补全隐藏（仅 subagent）|
| `disable` | 禁用代理 |
| `color` | UI 颜色（十六进制或主题颜色名）|
| 其他 | 作为模型选项传递（如 `reasoningEffort`、`textVerbosity`）|

### 8.7 子代理调用方式

**自动调用**：主代理根据 `description` 自动选择合适的子代理。

**手动调用**：用 `@` 提及：

```
@general help me search for this function
@explore find all callers of parseConfig
```

### 8.8 会话间导航

| 快捷键 | 说明 |
|--------|------|
| `<Leader>+Right` | 下一个子会话 |
| `<Leader>+Left` | 上一个子会话 |
| `<Leader>+Up` | 父会话 |
| `<Leader>+Down` | 第一个子会话 |

### 8.9 Task 工具权限控制

控制代理可通过 Task 工具调用哪些子代理：

```jsonc
{
  "agent": {
    "orchestrator": {
      "mode": "primary",
      "permission": {
        "task": {
          "*": "deny",
          "orchestrator-*": "allow",
          "code-reviewer": "ask"
        }
      }
    }
  }
}
```

### 8.10 opencode agent create/list 命令

```bash

```

`create` 命令交互式引导创建新代理，生成配置文件。

## 9 插件与 Hooks

OpenCode 没有独立的 "Hooks" 系统，**hooks 功能通过插件系统实现**。插件用 TypeScript 编写，可监听各种事件。

### 9.1 插件位置

| 位置 | 用途 |
|------|------|
| `.opencode/plugins/` | 项目级插件 |
| `~/.config/opencode/plugins/` | 全局级插件 |
| npm 包 | 在配置中 `"plugin": ["plugin-name"]` |

### 9.2 插件结构

```typescript
import type { Plugin } from "@opencode-ai/plugin"

export const MyPlugin: Plugin = async ({ project, client, $, directory, worktree }) => {
  return {
    // 钩子实现
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        // 修改 bash 命令
      }
    },
    "tool.execute.after": async (input, output) => {
      // 工具执行后处理
    },
    event: async ({ event }) => {
      if (event.type === "session.idle") {
        // 会话完成时通知
      }
    }
  }
}
```

### 9.3 事件类型完整列表

| 类别 | 事件 |
|------|------|
| 命令 | `command.executed` |
| 文件 | `file.edited`、`file.watcher.updated` |
| 安装 | `installation.updated` |
| LSP | `lsp.client.diagnostics`、`lsp.updated` |
| 消息 | `message.part.removed`、`message.part.updated`、`message.removed`、`message.updated` |
| 权限 | `permission.asked`、`permission.replied` |
| 服务器 | `server.connected` |
| 会话 | `session.created`、`session.compacted`、`session.deleted`、`session.diff`、`session.error`、`session.idle`、`session.status`、`session.updated` |
| 待办事项 | `todo.updated` |
| Shell | `shell.env` |
| 工具 | `tool.execute.after`、`tool.execute.before` |
| TUI | `tui.prompt.append`、`tui.command.execute`、`tui.toast.show` |
| 实验性 | `experimental.session.compacting`（压缩钩子）|

### 9.4 tool.execute.before / after 实战

**拦截 bash 命令**：

```typescript
import type { Plugin } from "@opencode-ai/plugin"

export const BashGuard: Plugin = async () => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        const cmd = input.input.command
        // 拦截危险命令
        if (cmd.includes("rm -rf /")) {
          throw new Error("Destructive command blocked")
        }
        // 记录命令
        console.log(`[BashGuard] executing: ${cmd}`)
      }
    },
    "tool.execute.after": async (input, output) => {
      if (input.tool === "bash") {
        console.log(`[BashGuard] completed: ${input.input.command}`)
      }
    }
  }
}
```

### 9.5 session.idle 通知实战

**会话完成时发送通知**：

```typescript
import type { Plugin } from "@opencode-ai/plugin"

export const Notifier: Plugin = async ({ $ }) => {
  return {
    event: async ({ event }) => {
      if (event.type === "session.idle") {
        // macOS 通知
        await $`osascript -e 'display notification "Task completed" with title "OpenCode"'`
      }
    }
  }
}
```

### 9.6 npm 插件安装与缓存

在配置中声明 npm 插件：

```jsonc
{
  "plugin": ["opencode-helicone-session"]
```

## 10 Skills

Skills 是 OpenCode 的可复用指令包，通过 `SKILL.md` 定义。OpenCode 兼容 Claude Code 和 agents.md 的 skills 标准。

### 10.1 SKILL.md 目录结构

为每个技能创建文件夹，放入 `SKILL.md`：

```
my-skill/
└── SKILL.md
```

### 10.2 frontmatter 字段

```markdown
---
name: git-release
description: Create consistent releases and changelogs
license: MIT
compatibility: opencode
metadata:
  audience: maintainers
  workflow: github
---

## What I do
- Draft release notes from merged PRs
- Propose a version bump
```

| 字段 | 说明 |
|------|------|
| `name`（必填）| 1-64 字符，小写字母数字+连字符，正则 `^[a-z0-9]+(-[a-z0-9]+)*$` |
| `description`（必填）| 1-1024 字符 |
| `license`（可选）| 许可证 |
| `compatibility`（可选）| 兼容性标记 |
| `metadata`（可选）| 字符串映射 |

### 10.3 发现机制

**项目本地**：从当前工作目录向上遍历到 git 工作树根目录。

**全局**：从以下目录加载：

| 范围 | 位置 |
|------|------|
| OpenCode | `~/.config/opencode/skills/` |
| Claude 兼容 | `~/.claude/skills/` |
| Agent 兼容 | `~/.agents/skills/` |

### 10.4 兼容 .claude/skills 与 .agents/skills

OpenCode 主动兼容 Claude Code 的 skills 目录：

- `.claude/skills/<name>/SKILL.md`
- `~/.claude/skills/<name>/SKILL.md`
- `.agents/skills/<name>/SKILL.md`
- `~/.agents/skills/<name>/SKILL.md`

这意味着从 Claude Code 迁移到 OpenCode 时，<span style="color:red;font-weight:bold;">skills 无需迁移</span>，OpenCode 自动发现。

### 10.5 权限控制

```jsonc
{
  "permission": {
    "skill": {
      "*": "allow",
      "pr-review": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
```

### 10.6 禁用 skill 工具

为特定代理禁用 skill 工具：

```jsonc
{
  "agent": {
    "plan": {
      "tools": { "skill": false }
    }
  }
}
```

### 10.7 显式调用与隐式调用

**隐式调用**：OpenCode 根据 `description` 自动匹配并加载 skill。

**显式调用**：通过原生 `skill` 工具调用：

```
用 git-release skill 为这次发布生成 release notes
```

## 11 自定义工具与命令

OpenCode 支持自定义工具（TypeScript）和自定义 Slash 命令（JSON/Markdown）。

### 11.1 自定义工具

在 `.opencode/tools/` 或 `~/.config/opencode/tools/` 中创建 TypeScript/JavaScript 文件：

```typescript
// .opencode/tools/query-db.ts
import { tool } from "@opencode-ai/plugin"

export default tool({
  description: "Query the project database",
  args: {
    query: tool.schema.string().describe("SQL query")
  },
  async execute(args, context) {
    const { agent, sessionID, messageID, directory, worktree } = context
    // 执行查询
    return `Executed query: ${args.query}`
  }
})
```

### 11.2 tool() 函数与 schema

```typescript
export default tool({
  description: "...",    // 工具描述
  args: {                // 参数 schema
    query: tool.schema.string().describe("SQL query"),
    limit: tool.schema.number().optional().describe("Max results")
  },
  async execute(args, context) {
    return "result"
  }
})
```

`tool.schema` 基于 Zod，支持 `string()`、`number()`、`boolean()`、`array()`、`object()` 等。

### 11.3 工具 context 参数

```typescript
async execute(args, context) {
  const { agent, sessionID, messageID, directory, worktree } = context
  // agent：当前代理名
  // sessionID：会话 ID
  // messageID：消息 ID
  // directory：当前工作目录
  // worktree：git 工作树路径
}
```

Anthropic 模型的思考预算：

```jsonc
{
  "provider": {
    "anthropic": {
      "models": {
        "claude-sonnet-4-5": {
          "options": {
            "thinking": {
              "type": "enabled",
```

`budgetTokens` 越高，模型思考越深入，但消耗更多 token。

### 18.5 迁移指南

#### CLAUDE.md -> AGENTS.md（兼容回退，无需立即迁移）

OpenCode 兼容 `CLAUDE.md`，无需立即重命名：

```bash
# 保持现状，OpenCode 自动回退读取 CLAUDE.md
$ opencode

# 或重命名（推荐，长期迁移）
$ mv CLAUDE.md AGENTS.md
```

#### settings.json -> opencode.json（格式 + 字段映射）

```json
// settings.json (Claude)
{
  "model": "sonnet",
  "env": {
    "ANTHROPIC_API_KEY": "sk-ant-xxx"
  }
}
```

```jsonc
// opencode.json (OpenCode)
{
  "model": "anthropic/claude-sonnet-4-5",
  "provider": {
    "anthropic": {
      "options": {
        "apiKey": "{env:ANTHROPIC_API_KEY}"
      }
    }
  }
}
```

#### .claude/skills/ 兼容（无需迁移）

OpenCode 自动发现 `.claude/skills/` 和 `~/.claude/skills/`，无需迁移。

```bash
# 保持现状
$ ls .claude/skills/
my-skill/

$ opencode
# OpenCode 自动加载 .claude/skills/my-skill/
```

#### .claude/agents/ -> .opencode/agents/（.json -> .md）

格式从 JSON 转 Markdown frontmatter：

```json
// .claude/agents/reviewer.json (Claude)
{
  "name": "reviewer",
  "description": "PR reviewer",
  "tools": ["Read", "Grep"]
}
```

```markdown
<!-- .opencode/agents/reviewer.md (OpenCode) -->
---
description: PR reviewer
mode: subagent
model: anthropic/claude-sonnet-4-5
tools:
  read: true
  grep: true
  edit: false
---

You are a PR reviewer. Focus on correctness and security.
```

#### hooks -> 插件（重构为 TypeScript 插件）

Claude Code 的 hooks 需重构为 OpenCode 插件：

```typescript
// .opencode/plugins/my-hooks.ts
import type { Plugin } from "@opencode-ai/plugin"

export const MyHooks: Plugin = async () => {
  return {
    "tool.execute.before": async (input, output) => {
      // 原 PreToolUse hook 逻辑
    },
    "tool.execute.after": async (input, output) => {
      // 原 PostToolUse hook 逻辑
    },
    event: async ({ event }) => {
      if (event.type === "session.idle") {
        // 原 Stop hook 逻辑
      }
    }
  }
}
```

事件映射：

| Claude Code | OpenCode |
|-------------|----------|
| `PreToolUse` | `tool.execute.before` |
| `PostToolUse` | `tool.execute.after` |
| `Stop` | `session.idle` |
| `SessionStart` | `session.created` |
| `UserPromptSubmit` | `message.updated` |

### 18.6 兼容性开关

完全禁用所有 Claude Code 兼容性：

```bash
$ export OPENCODE_DISABLE_CLAUDE_CODE=1
$ opencode
```

| 环境变量 | 说明 |
|----------|------|
| `OPENCODE_DISABLE_CLAUDE_CODE` | 禁用所有 .claude 支持 |
| `OPENCODE_DISABLE_CLAUDE_CODE_PROMPT` | 仅禁用 `~/.claude/CLAUDE.md` |
| `OPENCODE_DISABLE_CLAUDE_CODE_SKILLS` | 仅禁用 `.claude/skills` |

### 18.7 哲学差异

**OpenCode**：
- **开放生态**：开源、模型无关、75+ Provider
- **多客户端**：TUI、Web、Desktop、IDE 共享后端
- **Zen 网关**：精选模型，降低入门门槛
- **兼容性强**：主动兼容 Claude Code、agents.md 标准
- **企业就绪**：SSO、自托管、集中配置

**Claude Code**：
- **单一生态**：Anthropic 官方、绑定 Claude 模型
- **轻量简洁**：单一 CLI，依赖开发者自律
- **context management**：CLAUDE.md、subagents、skills
- **原生集成**：与 Anthropic API 深度优化

### 18.8 选型建议

**选 OpenCode 的场景**：

- 想用多个 LLM 提供商（不限于 Claude）
- 需要本地模型（Ollama、LM Studio）
- 想要 Web/桌面应用
- 需要 HTTP API 或 SDK 集成
- 团队需要 BYOK（自带密钥）
- 想要内置 LSP 和格式化工具
- 想要开源方案

**选 Claude Code 的场景**：

- 已订阅 Claude Pro/Max
- 主要用 Claude 模型
- 喜欢 `CLAUDE.md` + `settings.json` 的简单配置
- 不需要多客户端
- 想要与 Anthropic API 深度优化

**两者并用**：

- 不同项目用不同工具
- 利用 OpenCode 的 `CLAUDE.md` 兼容性，平滑迁移
- OpenCode 做多模型对比，Claude Code 做深度 Claude 开发
