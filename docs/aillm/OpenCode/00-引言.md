## 0 前置条件

### 0.1 系统要求

| 要求 | 详情 |
|------|------|
| 终端 | 现代终端模拟器（WezTerm、Alacritty、Ghostty、Kitty 等）|
| API 密钥 | 至少一个 LLM 提供商的 API 密钥（或用 OpenCode Zen）|
| 操作系统 | macOS、Linux、Windows（推荐 WSL）|

> **注意**：OpenCode 是开源的 AI 编码代理（open source AI coding agent），基于 AI SDK 和 Models.dev，支持 75+ LLM 提供商。与 Claude Code（绑定 Anthropic 模型）和 Codex（绑定 OpenAI 模型）不同，OpenCode 不绑定特定模型，可使用任意 LLM 提供商，包括本地模型（Ollama、LM Studio、llama.cpp）。

### 0.2 OpenCode 是什么

OpenCode 是一个开源的 AI 编码代理，核心特性：

- **模型无关**：支持 75+ LLM 提供商（Anthropic、OpenAI、Google、Bedrock、Azure、OpenRouter、GitHub Copilot 等）+ 本地模型
- **多客户端**：TUI（终端）、Web 界面、桌面应用（Beta）、IDE 扩展
- **OpenCode Zen**：自己的精选模型网关服务，推荐入门
- **兼容 Claude Code**：支持 `CLAUDE.md` 作为规则文件回退、`.claude/skills/` 技能目录
- **开放生态**：支持 MCP、Skills、自定义工具、插件、LSP、格式化工具

## 1 安装 OpenCode

安装和配置 OpenCode 非常简单，详细步骤请参考官方文档：

https://opencode.ai/docs/zh-cn/installation

### 1.1 官方安装脚本（推荐）

```bash
$ curl -fsSL https://opencode.ai/install | bash
```

安装路径优先级：

1. `$OPENCODE_INSTALL_DIR`
2. `$XDG_BIN_DIR`
3. `$HOME/bin`
4. `$HOME/.opencode/bin`

### 1.2 Homebrew

```bash
# 推荐，最新版本
$ brew install anomalyco/tap/opencode

# 官方 formula，更新较慢
$ brew install opencode
```

### 1.3 npm

```bash
$ npm install -g opencode-ai

# 也支持其他包管理器
$ bun install -g opencode-ai
$ pnpm install -g opencode-ai
$ yarn global add opencode-ai
```

### 1.4 其他方式

**Arch Linux**：

```bash
$ sudo pacman -S opencode        # 稳定版
$ paru -S opencode-bin           # AUR 最新版
```

**Windows**：

- <span style="color:red;font-weight:bold;">推荐 WSL</span>（最佳体验）
- Chocolatey: `choco install opencode`
- Scoop: `scoop install opencode`
- Docker: `docker run -it --rm ghcr.io/anomalyco/opencode`

**Nix**：

```bash
$ nix run nixpkgs#opencode
```

### 1.5 桌面应用（Beta）

从 https://opencode.ai/download 或 GitHub Releases 下载。支持：

- macOS（Apple Silicon / Intel）
- Windows
- Linux（.deb / .rpm / .AppImage）

### 1.6 验证与更新

```bash
$ opencode --version
opencode 0.20.x
```

更新到最新版本：

```bash
$ opencode upgrade
```

自动更新配置：

```jsonc
// ~/.config/opencode/opencode.json
{
  "autoupdate": true       // true（自动更新）| false（禁用）| "notify"（仅通知）
}
```

## 2 登录认证

OpenCode 支持多种认证方式，选择合适的方式直接影响可用 Provider 和计费。

### 2.1 认证方式对比

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| API Key | 大多数提供商的标准认证 | 个人开发、CI/CD |
| OAuth | Anthropic Claude Pro/Max、GitHub Copilot、OpenAI ChatGPT Plus/Pro、GitLab Duo | 订阅用户 |
| 环境变量 | 如 `ANTHROPIC_API_KEY` | 脚本、CI/CD |
| 项目 `.env` | 项目根目录 `.env` 文件 | 项目级配置 |
| OpenCode Zen | 精选模型网关，按请求付费 | 推荐入门 |

### 2.2 opencode auth 命令

```bash
$ opencode auth login       # 添加提供商 API 密钥
$ opencode auth list        # 列出已认证提供商（别名 opencode auth ls）
$ opencode auth logout      # 清除凭据
```

### 2.3 TUI 内 /connect 认证

在 TUI 中运行 `/connect`，选择提供商，粘贴 API 密钥：

```
/connect
# 选择提供商（如 anthropic）
# 粘贴 API 密钥
# 认证完成
```

### 2.4 OpenCode Zen（推荐入门）

OpenCode Zen 是 OpenCode 团队测试验证的精选模型网关：

- 在 TUI 中 `/connect` 选择 `opencode`
- 前往 opencode.ai/auth 获取密钥
- 按请求付费，支持自动充值（余额低于 $5 时自动充值 $20）
- 支持月度使用限额
- 团队功能：Admin/Member 角色、模型访问控制、自带密钥（BYOK）

```bash
$ opencode auth login
# 选择 opencode
# 粘贴 Zen 密钥
```

### 2.5 OAuth 提供商

支持 OAuth 登录的提供商：

| 提供商 | 订阅要求 |
|--------|----------|
| Anthropic | Claude Pro / Max |
| OpenAI | ChatGPT Plus / Pro |
| GitHub Copilot | Copilot 订阅 |
| GitLab Duo | Duo 订阅 |

OAuth 流程会自动打开浏览器完成认证。

### 2.6 环境变量认证

```bash
# Anthropic
$ export ANTHROPIC_API_KEY=sk-ant-xxx

# OpenAI
$ export OPENAI_API_KEY=sk-xxx

# 启动
$ opencode
```

### 2.7 凭据存储位置

| 平台 | 路径 |
|------|------|
| macOS / Linux | `~/.local/share/opencode/auth.json` |
| Windows | `%USERPROFILE%\.local\share\opencode\auth.json` |

## 3 opencode 命令速查

### 3.1 常用命令

```bash
$ opencode                    # 启动 TUI（默认）
$ opencode /path/to/project   # 指定工作目录启动 TUI
$ opencode run "提示词"        # 非交互模式运行
$ opencode serve              # 启动无界面 HTTP 服务器
$ opencode web                # 启动带 Web 界面的服务器
$ opencode attach [url]       # 连接到已运行的 OpenCode 后端
$ opencode auth login         # 添加提供商 API 密钥
$ opencode mcp list           # 列出 MCP 服务器
$ opencode models             # 列出可用模型
$ opencode agent list         # 列出所有代理
$ opencode session list       # 列出会话
$ opencode stats              # 显示 Token 用量和费用统计
$ opencode upgrade            # 更新到最新版本
$ opencode uninstall          # 卸载 OpenCode
```

### 3.2 子命令完整清单

| 子命令 | 说明 |
|--------|------|
| `opencode [project]`（tui）| 启动 TUI |
| `opencode run [message..]` | 非交互模式运行 |
| `opencode serve` | 启动无界面 HTTP 服务器 |
| `opencode web` | 启动带 Web 界面的服务器 |
| `opencode attach [url]` | 连接到已运行的 OpenCode 后端 |
| `opencode acp` | 启动 ACP（Agent Client Protocol）服务器 |
| `opencode agent [command]` | 管理代理（create、list）|
| `opencode auth [command]` | 管理凭据（login、list、logout）|
| `opencode mcp [command]` | 管理 MCP 服务器（add、list、auth、logout、debug）|
| `opencode models [provider]` | 列出可用模型 |
| `opencode session [command]` | 管理会话（list）|
| `opencode stats` | 显示 Token 用量和费用统计 |
| `opencode export [sessionID]` | 导出会话为 JSON |
| `opencode import <file>` | 导入会话 |
| `opencode github [command]` | 管理 GitHub 集成（install、run）|
| `opencode upgrade [target]` | 更新到最新或指定版本 |
| `opencode uninstall` | 卸载 OpenCode |

### 3.3 常用全局 flag

| Flag | 说明 |
|------|------|
| `--model` / `-m` | 指定模型（provider/model 格式，如 `anthropic/claude-sonnet-4-5`）|
| `--continue` / `-c` | 继续上一个会话 |
| `--session` / `-s` | 指定会话 ID |
| `--fork` | 分叉会话 |
| `--prompt` | 使用的提示词 |
| `--agent` | 指定代理 |
| `--port` / `--hostname` | 监听端口和主机名（serve/web）|
| `--print-logs` | 日志输出到 stderr |
| `--log-level` | 日志级别（DEBUG、INFO、WARN、ERROR）|
| `--help` / `-h` | 帮助 |
| `--version` / `-v` | 版本 |

### 3.4 run 命令特有 flag

| Flag | 说明 |
|------|------|
| `--command` | 要运行的命令 |
| `--share` | 分享会话 |
| `--file` / `-f` | 附加文件 |
| `--format` | `default` 或 `json` |
| `--title` | 会话标题 |
| `--attach` | 连接到运行中的服务器 |
| `--variant` | 模型变体（推理级别）|
| `--thinking` | 显示思考块 |

### 3.5 stats 命令示例

```bash
# 查看最近 30 天、Top 5 模型的用量
$ opencode stats --days 30 --models 5

# 查看当前项目用量
$ opencode stats --project ""
```

## 4 内置 Slash 命令列表

在 OpenCode 交互式 TUI 中，以 `/` 开头的命令用于快速操作。大多数命令有 `ctrl+x` 前导键快捷方式。

### 4.1 核心 Slash 命令

| 命令 | 快捷键 | 说明 | 使用频率 |
|------|--------|------|----------|
| `/help` | `ctrl+x h` | 显示帮助 | ⭐⭐⭐⭐⭐ 最高 |
| `/new` | `ctrl+x n` | 开始新会话（别名 `/clear`）| ⭐⭐⭐⭐⭐ 最高 |
| `/sessions` | `ctrl+x l` | 列出会话并切换（别名 `/resume`、`/continue`）| ⭐⭐⭐⭐⭐ 最高 |
| `/models` | `ctrl+x m` | 列出可用模型 | ⭐⭐⭐⭐⭐ 最高 |
| `/compact` | `ctrl+x c` | 压缩当前会话（别名 `/summarize`）| ⭐⭐⭐⭐⭐ 最高 |
| `/init` | `ctrl+x i` | 创建/更新 AGENTS.md | ⭐⭐⭐⭐ 高（每项目一次）|
| `/connect` | - | 添加提供商 API 密钥 | ⭐⭐⭐⭐ 高（首次配置）|
| `/thinking` | - | 切换思考块显示 | ⭐⭐⭐⭐ 高 |
| `/undo` | `ctrl+x u` | 撤销最后一条消息 | ⭐⭐⭐ 中 |
| `/redo` | `ctrl+x r` | 重做撤销的消息 | ⭐⭐⭐ 中 |
| `/details` | `ctrl+x d` | 切换工具执行详情显示 | ⭐⭐⭐ 中 |
| `/editor` | `ctrl+x e` | 打开外部编辑器 | ⭐⭐ 低 |
| `/export` | `ctrl+x x` | 导出对话为 Markdown | ⭐⭐ 低 |

### 4.2 前导键（Leader Key）机制

OpenCode 大多数快捷键需要先按前导键 `ctrl+x`，再按对应键。前导键超时默认 2000ms，可在 `tui.json` 中配置：

```jsonc
// ~/.config/opencode/tui.json
{
  "leader_timeout": 2000    // 前导键超时（毫秒）
}
```
