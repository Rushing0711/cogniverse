## 0 前置条件

### 0.1 系统要求

| 要求 | 详情 |
|------|------|
| 操作系统 | macOS 12+、Ubuntu 20.04+/Debian 10+、Windows 11（**必须通过 WSL2**）|
| Git（可选，推荐）| 2.23+，用于内置 PR helpers |
| 内存 | 最低 4GB，推荐 8GB |

> **注意**：Codex CLI 本体是 Rust 写的原生二进制（仓库下 `codex-rs/`），不是 Node.js 程序；npm 包只是发行包装。这与 Claude Code（Node.js）有本质不同，启动更快、内存占用更低。

### 0.2 Git

需要安装 Git 命令，用于内置 PR helpers、worktree 管理等：

```bash
$ git --version
git version 2.44.1
```

## 1 安装 Codex

安装和配置 Codex 非常简单，详细步骤请参考官方文档：

https://developers.openai.com/codex/cli

### 1.1 官方安装脚本（推荐）

**macOS / Linux**：

```bash
$ curl -fsSL https://chatgpt.com/codex/install.sh | sh
```

**Windows（PowerShell）**：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"
```

安装脚本支持两个环境变量：

| 环境变量 | 作用 |
|----------|------|
| `CODEX_NON_INTERACTIVE=1` | 跳过所有交互式提示，用于 CI |
| `CODEX_INSTALL_DIR` | 自定义安装位置（默认 `~/.local/bin` 或 `%LOCALAPPDATA%\Programs\OpenAI\Codex\bin`）|

非交互式安装示例（CI 场景）：

```bash
$ curl -fsSL https://chatgpt.com/codex/install.sh | CODEX_NON_INTERACTIVE=1 sh
```

### 1.2 Homebrew

```bash
$ brew install --cask codex
```

### 1.3 npm

```bash
$ npm install -g @openai/codex
```

> **注意**：npm 包只是 Rust 二进制的发行包装，仍需系统支持原生二进制运行。Windows 用户需在 WSL2 环境下安装。

### 1.4 GitHub Releases 二进制下载

地址：https://github.com/openai/codex/releases/latest

平台对应文件：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon | `codex-aarch64-apple-darwin.tar.gz` |
| macOS x86_64 | `codex-x86_64-apple-darwin.tar.gz` |
| Linux x86_64 | `codex-x86_64-unknown-linux-musl.tar.gz` |
| Linux arm64 | `codex-aarch64-unknown-linux-musl.tar.gz` |

每个压缩包内是带平台后缀的单文件二进制，解压后需重命名为 `codex` 并放到 `PATH` 中。

### 1.5 验证与诊断

```bash
$ codex --version
codex 0.143.0
```

生成诊断报告（包含本地安装、配置、认证、运行时、Git、终端、app-server、线程等清单问题）：

```bash
$ codex doctor
```

`codex doctor` 是排查问题的第一工具，遇到任何异常先跑一遍。

### 1.6 自更新

```bash
$ codex update
```

当安装版本支持自更新时，会检查并应用更新。若想关闭启动时的更新检查：

```toml
# ~/.codex/config.toml
check_for_update_on_startup = false
```

## 2 登录认证

Codex 支持多种登录方式，选择合适的方式直接影响计费和可用功能。

### 2.1 两种登录方式对比

| 维度 | Sign in with ChatGPT | Sign in with API Key |
|------|----------------------|---------------------|
| 计费 | Plus/Pro/Business/Edu/Enterprise 套餐额度 | 标准 API 用量计费（按 token）|
| 适用 | 个人/团队订阅用户 | CI/CD、程序化工作流 |
| 可用功能 | 全部（包括 Codex Cloud、Workspace RBAC 等）| 仅本地工作流，部分功能受限 |
| Fast Mode | 可用（信用消费倍率）| 不可用，走标准 API 计价 |
| 管理控制 | ChatGPT workspace 权限、RBAC、企业保留期 | API 组织保留期与数据共享设置 |

### 2.2 ChatGPT OAuth 登录（推荐）

```bash
$ codex login
```

- 弹出浏览器，完成 OAuth 流程后回调本地
- 凭据缓存到 `~/.codex/auth.json`（明文文件）或 OS keychain/keyring
- Token 自动刷新，活跃会话通常不需要重新登录

### 2.3 API Key 登录

```bash
$ printenv OPENAI_API_KEY | codex login --with-api-key
```

API Key 用量走 OpenAI Platform 账户，按标准 API 价格计费。适合 CI/CD、脚本自动化场景。

### 2.4 Device Code 登录（无头/远程环境，Beta）

适用场景：远程或无头环境、本地网络阻止 localhost 回调。

```bash
$ codex login --device-auth
```

需要在 ChatGPT 安全设置中预先启用 device code 登录。流程：

1. CLI 打印一个链接和一次性代码
2. 浏览器登录并输入代码
3. CLI 收到 token

### 2.5 Codex Access Token（企业自动化）

ChatGPT Enterprise 工作区管理员可授予 access token 权限，让成员为受信任的非交互式本地工作流创建 access token：

```bash
$ printenv CODEX_ACCESS_TOKEN | codex login --with-access-token
```

适用于需要 ChatGPT workspace 访问、企业控制但无浏览器登录的脚本、调度器、私有 CI runner。

### 2.6 凭据存储位置

`cli_auth_credentials_store` 配置项控制凭据存储位置：

| 值 | 行为 |
|----|------|
| `file` | 存储在 `~/.codex/auth.json`（明文）|
| `keyring` | 存储在 OS 凭据存储（macOS Keychain / Windows Credential Manager / Linux Secret Service）|
| `auto`（默认）| 优先用 OS 凭据存储，不可用时回退到文件 |

```toml
# ~/.codex/config.toml
cli_auth_credentials_store = "keyring"
```

### 2.7 检查与登出

```bash
$ codex login status    # 查看当前认证方式
$ codex logout          # 清除当前凭据
```

### 2.8 强制登录方式（企业管理）

企业环境可强制限定登录方式：

```toml
# ~/.codex/config.toml

# 仅允许 ChatGPT 登录或仅允许 API Key 登录
forced_login_method = "chatgpt"   # 或 "api"

# 限制 ChatGPT 登录到特定 workspace
forced_chatgpt_workspace_id = "00000000-0000-0000-0000-000000000000"
```

### 2.9 企业 CA 证书

如果公司网络有 TLS 拦截或私有 CA：

```bash
$ export CODEX_CA_CERTIFICATE=/path/to/corporate-root-ca.pem
$ codex login
```

未设 `CODEX_CA_CERTIFICATE` 时回退到 `SSL_CERT_FILE`。

### 2.10 无头环境登录 fallback

远程服务器无浏览器时，按优先级选择：

1. **首选**：Device Code 认证（见 2.4）
2. **Fallback 1**：本地登录后复制 `~/.codex/auth.json` 到远程机器
   ```bash
   $ ssh user@remote 'mkdir -p ~/.codex'
   $ scp ~/.codex/auth.json user@remote:~/.codex/auth.json
   ```
3. **Fallback 2**：SSH 端口转发
   ```bash
   $ ssh -L 1455:localhost:1455 user@remote
   # 在 SSH 会话中运行 codex login
   ```

## 3 codex 命令速查

### 3.1 常用命令

```bash
$ codex --help              # 查看帮助
$ codex                     # 启动交互式 TUI（默认）
$ codex exec "..."          # 非交互式运行，结果流到 stdout
$ codex resume              # 继续之前的交互式 session
$ codex fork                # 分叉之前的 session 为新 task
$ codex review              # 非交互式代码审查
$ codex mcp list            # 列出已配置 MCP 服务器
$ codex mcp add <name> -- <command> [args...]   # 添加 MCP 服务器
$ codex sandbox macos       # 在沙箱中测试命令（macOS）
$ codex doctor              # 生成诊断报告
$ codex update              # 检查并应用更新
$ codex login status        # 查看认证状态
$ codex plugin list         # 列出已安装插件
$ codex completion zsh      # 生成 shell 补全脚本
```

### 3.3 常用全局 flag
